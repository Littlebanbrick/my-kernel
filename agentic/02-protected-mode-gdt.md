# 学习笔记：进入 32 位保护模式

> 日期：2026-06-30
> 目标：从 16 位实模式切换到 32 位保护模式，并用 VGA 显存直接输出（脱离 BIOS）

---

## 一、保护模式核心概念

### 实模式 vs 保护模式

| | 实模式 | 保护模式 |
|--|--------|----------|
| CPU 位宽 | 16 位 | **32 位** |
| 最大内存 | 1 MB | **4 GB** |
| 段寻址方式 | `CS × 16 + IP` | GDT 描述符（基址 + 限长 + 权限） |
| 权限检查 | 无 | Ring 0 ~ Ring 3，越界/越权→异常 |
| BIOS 中断 | 可用 | **不可用** |

### 保护模式的核心——GDT（全局描述符表）

GDT 是内存中的一张表，每个条目 **8 字节**，描述一个"段"的属性：

```
一个 GDT 描述符（8 字节）：
┌────────┬────────┬────────┬────────┬────────┬────────┬────────┬────────┐
│ 限长    │ 基址    │ 基址    │ 访问    │ 标志+   │ 基址    │
│ 低16位  │ 低16位  │ 16-23位│ 权限    │ 限长高4 │ 24-31位│
│        │        │        │ 字节    │ 位      │        │
└────────┴────────┴────────┴────────┴────────┴────────┴────────┴────────┘
byte 0-1  byte 2-3 byte 4  byte 5  byte 6  byte 7
```

访问权限字节（决定段的行为）：

| 位 | 含义 |
|----|------|
| bit 7 (Pr) | Present = 1（段有效） |
| bit 6-5 (DPL) | 特权级（00=Ring 0 内核，11=Ring 3 用户） |
| bit 4 (S) | 1=代码/数据段，0=系统段 |
| bit 3 (Ex) | 1=代码段（可执行），0=数据段 |
| bit 2 (DC) | 代码：Conforming；数据：Expand-down |
| bit 1 (RW) | 代码：Readable；数据：Writable |
| bit 0 (Ac) | Accessed（CPU 自动置位） |

### 选择子（Selector）

保护模式的段寄存器不存基址，而是存一个 **GDT 索引**：

```
CS = 0x08  →  二进制 0000 0000 0000 1000
                ↑        └──┴─┘
                │           ↑ 索引 = 1（GDT[1]，代码段）
                └─── TI = 0（使用 GDT）
                └──── RPL = 0（Ring 0）
```

### 平坦内存模型

让所有段寄存器指向**同一个** GDT 条目（基址 = 0，限长 = 4 GB）：

```
DS = 0x10 → GDT[2]: base=0, limit=4GB
ES = 0x10 → GDT[2]: base=0, limit=4GB
SS = 0x10 → GDT[2]: base=0, limit=4GB
```

这样你写的**任何 32 位地址直接等于物理地址**，不用再操心段的切换。

### 从实模式切换到保护模式的完整序列

```
① LGDT    加载 GDT 伪描述符到 GDTR 寄存器
    GDTR = { 基址: GDT 起始物理地址, 限长: 大小-1 }

② CR0.PE = 1
    设置控制寄存器 bit 0，CPU 立即进入保护模式

③ far jump（远跳转）
    用新选择子（如 0x08）加载 CS，刷新指令预取队列
    CPU 查 GDT[1] → 缓存描述符到 CS 隐藏部分

④ 重载 DS/ES/FS/GS/SS
    都指向数据段描述符（如 0x10），建立平坦内存模型
```

## 二、VGA 文本模式显存

### 内存映射

VGA 文本模式的显存映射在物理地址 **0xB8000**：

```
0xB8000:
┌────────┬────────┬────────┬────────┬────────┬────────┬─────
│ 字符  │ 属性   │ 字符  │ 属性   │ 字符  │ 属性   │ ...
├────────┼────────┼────────┼────────┼────────┼────────┼─────
│ col 0  │ col 0  │ col 1 │ col 1  │ col 2 │ col 2  │ ...
│ char   │ attr   │ char  │ attr   │ char  │ attr   │ ...
└────────┴────────┴────────┴────────┴────────┴────────┴─────
offset 0   off 1   off 2   off 3   off 4   off 5
```

每个字符格 = 2 字节：
- **低字节**：ASCII 字符码
- **高字节**：属性（前景色 + 背景色）

属性字节 0x07 = 0000 0111 = 黑底(0000) + 亮灰(0111)

### 行列计算公式

```
地址 = 0xB8000 + (行 × 80 + 列) × 2

例：第 0 行第 5 列 = 0xB8000 + (0×80+5)×2 = 0xB800A
```

### 清屏原理

```assembly
movl $0xB8000, %edi     # EDI = VGA 基地址
movl $2000, %ecx        # 80列×25行 = 2000 个字符格
movw $0x0720, %ax       # AX = 空格(0x20) + 白字黑底(0x07)
rep stosw               # 重复写 2000 次 = 全屏填满空格
```

`rep stosw`（Repeat Store String Word）等价于：
```c
while (ecx > 0) {
    *edi = ax;      // 写入一个字
    edi += 2;       // 前进到下一个字符格
    ecx--;
}
```

## 三、GDB 调试技巧

### 硬件断点 vs 软件断点

| 类型 | 命令 | 原理 | 适用场景 |
|------|------|------|---------|
| 软件断点 | `break` | 在内存写 0xCC（INT3） | 普通调试 |
| 硬件断点 | `hbreak` | 用 CPU 调试寄存器 | boot sector 调试（BIOS 加载时覆盖内存） |

### 跳过 BIOS 中断调用

```gdb
(gdb) nexti     # 一步跳过 int $0x10，不会陷入 BIOS 内部
```

### 跳过 `rep stosw` 等重复指令

```gdb
(gdb) break *0x7C48    # 在 rep stosw 后面设断点
(gdb) continue          # 一次执行完 2000 次写入
```

### 查看内存 / 寄存器

```gdb
(gdb) info registers      # 查看全部寄存器
(gdb) info registers cs ds cr0   # 只看指定寄存器
(gdb) x/8bx 0xB8000      # 查看 VGA 显存的原始字节
```

## 四、调试 GDT 相关问题的经验

### 1. DS 必须在 LGDT 前清零

`lgdt gdt_desc` 用 `DS:偏移量` 读取内存。BIOS 中断可能改了 DS → 读错地址 → GDTR 存入垃圾 → 远跳转入保护模式时查 GDT 无效 → triple fault。

**修复**：`xorw %ax, %ax; movw %ax, %ds`

### 2. 远跳转必须用 32 位偏移

`.code16` 模式默认生成 16 位远跳转。设置 CR0.PE 后 CPU 已在保护模式下，期望 32 位偏移。手动编码 `0x66 0xEA` + 32 位地址 + 16 位选择子。

### 3. VGA 写入目标位置是 0xB8000，不是光标位置

屏幕上显示的字符不是由"光标位置"决定的，而是由 **VGA 显存中对应地址的内容** 决定的。忘记这一点会让初学者以为代码没工作。

## 五、调试环境

- **QEMU 调试端**：使用 `-serial stdio` + 串口输出确认保护模式代码执行
- **QEMU 参数**：`-no-reboot` 让 triple fault 时 QEMU 退出（而不是重启），用来验证保护模式切换是否成功
