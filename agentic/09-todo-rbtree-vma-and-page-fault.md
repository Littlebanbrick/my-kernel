## TODO 1：虚拟地址空间管理（VMA + 红黑树）

**问题：** 当前使用 bump pointer 分配虚拟地址（永远从最后一块往后找），程序退出释放虚拟地址后产生空洞，无法复用。

**目标：** 用红黑树记录所有已占用的虚拟地址区间，分配时找到最小满足要求的连续空洞。

**前置：** 自学红黑树（插入、删除、查找、前驱/后继）。

**实现思路草图：**
- 每个节点 = `{ start, end, pid/flags }`
- 红黑树按 `start` 排序
- 找空洞：中序遍历找两个相邻节点之间的 gap 是否足够大
- 释放：删除节点，把释放的区间作为空洞

**关联：** 多进程实现后，每个进程有独立页目录 + 独立 VMA 红黑树。

## TODO 2：缺页处理（Page Fault Handler）

**问题：** 当前 #PF handler 直接 halt 系统。需要让它：

1. 从 CR2 读出触发缺页的虚拟地址
2. 查 VMA：该地址是否属于当前程序？
3. 如果属于 → 从 buddy 分配一页物理内存，`map_page` 映射到缺页地址，返回用户程序重新执行
4. 如果不属于 → 打印信息并 halt（杀进程的简化版）

**几种缺页场景待支持：**
- **栈自动增长：** 用户栈往下访问到未映射地址 → 分配新页
- **懒分配（demand paging）：** 程序申请内存时只记 VMA，不分配物理页。第一次访问时才补页
- **写时拷贝（COW）：** fork 后父子共享物理页，标记为只读。写操作触发 #PF → 复制页面

**当前状态：** #PF handler 入口已有（向量 14 走 `handle_exception`），但只会 halt。
